# Onboarding

> [!info] Why this page exists
> A solo founder doesn't need an onboarding doc. The first engineering hire does. This folder exists so onboarding is a checklist someone follows, not a series of ad-hoc conversations that vary by who happens to be free that week.

> [!tip] When to use this page
> Follow it, top to bottom, on someone's first days. Update it the moment a new hire hits a gap or a stale instruction — the best time to fix onboarding docs is while someone is actively experiencing the friction.

> [!note] How it connects to the rest of the vault
> It's the guided path through everything else: [[Vault Guide]] and [[Conventions]] for the vault itself, [[Canon]] for identity, [[Architecture Index]] and [[Systems Index]] for the technical map, [[Plugin Setup]] for tooling.
