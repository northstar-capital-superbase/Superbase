# Quarterly Plans

> [!info] Why this page exists
> [[Roadmap]] is a rolling, always-current view — useful day to day, but it doesn't capture "what did we commit to for this specific period, and did we hit it." This folder holds that dated, periodic record, which becomes valuable in retrospect precisely because it doesn't move.

> [!tip] When to use this page
> Create a new quarter's plan (from [[Quarterly Plan Template]]) at the start of each planning period. Look back at prior quarters here when assessing whether the team consistently over- or under-commits.

> [!note] How it connects to the rest of the vault
> A quarter's goals should trace back to [[Mission & Vision]] and pull specific items from [[Roadmap]]'s Now column. Once a quarter closes, its plan becomes historical record — don't edit it retroactively; note actual outcomes in a closing section instead.
