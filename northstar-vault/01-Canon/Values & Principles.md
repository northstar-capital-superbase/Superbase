# Values & Principles

> [!info] Why this page exists
> Values only matter if they change a decision when two good options conflict. This page exists to make those tie-breakers explicit and reusable, instead of re-litigating them in every [[ADR Index|ADR]] or [[Product Index|spec]] discussion.

> [!tip] When to use this page
> Cite it directly in an ADR's "Decision" or a spec's trade-off section when a value tips the balance ("we chose the slower rollout per [[Values & Principles#Safety|Safety]]"). Revisit during hiring/onboarding conversations and performance conversations once the team grows.

> [!note] How it connects to the rest of the vault
> [[04-Engineering/Standards/Standards Index|Engineering Standards]] should operationalize these values into concrete rules. [[ADR Index|ADRs]] should feel consistent with them; if an accepted decision seems to violate a value, that tension is worth writing down rather than ignoring.
